What is a wireframe?

Wireframes are indispensable tools in the field of Design. Typically used in Web, UI/UX, and Print Design, wireframes are sketches or blueprints created by interface designers to show the placement of elements on a page. They establish a set of visual guidelines for coders and developers to follow. They also establish the flow of a web page or app interface from start to finish, thereby building a narrative for the user experience. Jaye Hannah, editor for CareerFoundry, writes, “Wireframes are effectively used as communication devices; they facilitate feedback from the users, instigate conversations with the stakeholders, and generate ideas between the designers” (Hannah, 2019)¹. They thus make up an integral part of any front-end design process and are often used along with mockups, prototypes, and user personas.

Wireframes can be made using digital tools such as Adobe XD, Sketch, and Figma, or they can be drawn on paper. They do not have to be fully detailed - in fact, many wireframes contain only boxes and lines to show graphics and text. To create a wireframe, one must simply be able to mentally picture the desired interface and sketch (or digitally mockup) what they imagine. Clients then make notes on these sketches, telling the designer what to add or remove. Once a sketch is finalized, developers refer to them as they code the interface. This process is common and occurs in many different areas of Design - however, here I will be specifically addressing the application of wireframes in Web Design.

Types of Wireframes

Wireframes are of three main types: Low-fidelity wireframes, Mid-fidelity wireframes, and High-fidelity wireframes. Each of them deal with a different level of specificity in page content.


  • Low-fidelity wireframes: These are the most basic type, dealing only with page structure and element hierarchy. They are generally in grayscale and use only blocks and lines to mark up elements. Low-fidelity wireframes are not drawn to scale and are usually abstract, omitting any graphics, text, or font styles. If text is used, it is usually kept to a minimum and used only in places where it helps build understanding or lessen ambiguity, such as on a button. The purpose of a low-fidelity wireframe is to show clients the big picture of a webpage’s layout and purpose before fine-tuning the details.

  • Mid-fidelity wireframes: Having slightly more detail than low-fidelity wireframes, these wireframes generally use some placeholder (‘Lorem ipsum’) text instead of lines. Other graphic elements are still represented by boxes, and font styles are still omitted. These wireframes can be done in grayscale or using one spot color (a monochromatic color scheme). They are sometimes created using digital wireframing tools to keep the scale, margin, and sizing of elements consistent. Mid-fidelity wireframes are meant to give the client a more accurate blueprint of the interface so that they can decide what works and what doesn’t. Tsvetelina Lazarova, an Experience Designer, writes, “Medium fidelity wireframes can help to communicate to teams how aesthetic features can support essential functionalities” (Lazarova, 2018)².

  • High-fidelity wireframes: These are the most detailed kind of wireframes. They use some graphics and replace the filler text with actual website content. These are created using only digital software since the purpose is for the design to look professional. However, they must not be confused with prototypes. Prototypes show how an interface functions, while high-fidelity wireframes are still only concerned with interface design.
